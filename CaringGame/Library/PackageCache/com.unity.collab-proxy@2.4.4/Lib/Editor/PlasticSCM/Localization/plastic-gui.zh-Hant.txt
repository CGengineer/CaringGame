== PendingChangesSimilarityPercentage ==

== PendingChangesSimilarityPercentageExplanation ==
定義兩個案在比較後必須達到的「相似性程度」，才能偵測為「已移動」或「已重新命名」。例如，如果將 foo.c 移動到 bar.c，並於稍後進行修改，% 將定義這兩個案必須達到多少相似性，Unity VCS 才會將其視為相同案。
這個情況也適用於目：目結構必須達到多少相似性：相對於目總項目數有多少個已移動下層項目。

== PendingChangesEnableNTFSJournal ==

== PendingChangesEnableNTFSJournalExplanation ==
使用 NTFS 日誌，在 GUI 已開情況下偵測已移動的案與目。這種方法超級精確，而且沒有「可能性」或「發性」。NTFS 知道案或目的移動時間。

不過，這個做法有個缺點。如果專案重構時是使用像是 Visual Studio 等工具，使用這個功能就不安全。原因在於 VS 不會直接將 foo.c 移動到 src/bar.c，而是建立新的 src/bar.c，接著複製容，然後刪除 foo.c。因此無法從案系統進行追蹤。

如果打算使用 Windows 案總管或命令列進行大規模重構，這個功能將可流暢運作。
