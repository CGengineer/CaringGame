== CMD_DESCRIPTION_ACL ==
置象限。

== CMD_USAGE_ACL ==
用法：

    cm ^acl (--^user=<用名> | --^group=<名称>)
           (-^allowed|-^denied|-^overrideallowed|-^overridedenied=+|-<限>[,...])[,...]
           <象格>

    --^user             用名。
    --^group            名称。
    -^allowed           用指定的一或多限。使用
                       逗号各个限行分隔。（使用 'cm ^showpermissions'
                       可示所有可用限。）
    -^denied            拒指定的一或多限。使用
                       逗号各个限行分隔。（使用 'cm ^showpermissions'
                       可示所有可用限。）
    -^overrideallowed   覆盖允的一或多限。使用
                       逗号各个限行分隔。（使用 'cm ^showpermissions'
                       可示所有可用限。）
    -^overridedenied    覆盖拒的一或多限。使用
                       逗号各个限行分隔。（使用 'cm ^showpermissions'
                       可示所有可用限。）
    象格         要置限的象。
                       此命令的有效象：
                       存服器、存、分支、更集、、
                       和属性。
                       （使用 'cm ^help ^objectspec' 可一了解格。）

安全路径的特殊用法：
    cm ^acl [(--^user=<用名> | --^group=<名称>)
            (-^allowed|-^denied|-^overrideallowed|-^overridedenied=+|-<限>[,...])[,...]]
            [--^delete] [--^branches=[+ | -]<分支>[,...]]
            <格>

    --^delete           除安全路径。
                       参“注”以了解更多信息。
    --^branches         将安全路径限置一个分支。
                       使用逗号各个分支行分隔。
                       （可）个分支前面可添加 + 或 -
                       符号，从而指定在必在列表中添加
                       是除某个分支。
                       参“注”以了解更多信息。
    格               要置限的安全路径。

== CMD_HELP_ACL ==
了配置限，需要了解 Unity VCS 安全机制的工作原理。
看《安全指南》以了解限的工作原理：
https://www.plasticscm.com/download/help/securityguide

注：

    此命令置用或于指定象、存、分支、
    和/或服器路径的限。

    象格：
        （使用 'cm ^help ^objectspec' 可了解如何指定象。）
        '^acl' 命令使用一特殊型的格：安全路径。

        - 安全路径格：
            ^path:server_path[#tag]
            示例：^path:/src/foo.c
                      ^path:/doc/pdf
                      ^path:/doc/pdf#documents

    限操作：
        使用 -^allowed 和 -^denied 指定要置的限。
        使用 -^overrideallowed 和 -^overridedenied 参数指定
        要覆盖的限。

        个操作都需要一个限列表（以逗号分隔）。

    限名称：
        个限名称前面都有 + 或 - 符号。
        + 符号表示置限，而 - 符号表示清除限。
        要看象的限，使用 'cm ^showacl' 命令。

    覆盖的限：
        使用 -^overrideallowed 和 -^overridedenied
        来覆盖限可以承。
        于在存或服器置的限
        很有助。
        示例：
            cm ^acl --^user=vio -^allowed=+^ci -^overrideallowed=+^ci ^br:qa@test
            （允用 'vio' 在存 'test' 上的分支 'qa' 上入，
            即使用的限在存被拒也是如此。）

    服器路径限（又称“安全路径”）：
        - 允定的服器路径指定限。
        - 在入操作期会些限。
        - 也可以在更新操作期些限，
          并可以将些限用作一防止某些目和文件
          下到工作区的方法。
        - 于要入的一，服器都会将路径
          与安全路径匹配。如果匹配，入操作
          将是否可以入。

        可安全路径定的限
        如下：
            '^ci'、'^change'、'^add'、'^move'、'^rm'、'^read'

        于任何及的，如果限不成功，
        入操作将回。

        要将安全路径限置一个分支，使用
        --^branches 。
        示例：
          cm ^acl --^user=jo -^denied=+^ci ^path:/src#rule0 --^branches=main,main/rel0

        要与安全路径的 ACL，很有用。
        示例：
          cm ^acl --^user=jo -^denied=+^rm ^path:/src#rule0
          （如果没有，需要再次指定
          分支列表。）

        安全路径的分支列表是可以的。
        示例：
          cm ^acl ^path:/src#rule0 --^branches=-main,+main/rel1
          （从列表中除 'main' 并添加 'main/rel1'。）

        要除安全路径，使用 --^delete 参数。
        示例：
          cm ^acl --^user=jo --^delete ^path:/src#rule0

    承：
        承是 Plastic SCM 3.0 期的一。
        是高，但几乎已弃用。
        允象从任何其他象承限，
        从而覆盖默的承系。

        使用 -^cut 切断承。
        使用 -^cutncpy 切断并制当前承的
        限。（个受到 Windows 文件系限的，
        在 Windows 文件系中可以切断承系但保留限。）

        -^inherit 允用从象格承。
        示例：'-^inherit=象格'

示例：

    cm ^acl --^user=danipen -^denied=+^ci ^rep:core
    （拒用 'danipen' 在存 'core' 上入。）

    cm ^acl --^group=developers -^allowed=+^view,-^read -^denied=+^chgperm ^br:main
    （命令向 'main' 分支中的 'developers' 
    授予看限、清除取限并拒 chgperm 限。）

安全路径示例：

    cm ^acl --^group=devs -^denied=+^ci ^path:/server#rel --^branches=main,main/2.0
    （于在分支 'main' 和 'main/2.0' 中与 '/server' 匹配的任何路径，
    命令拒向 'devs' 授予入限。建 '#rel'
    是了稍后能行引用。）

    cm ^acl ^path:/server#rel --^branches=-/main,+/main/Rel2.1
    （更新具有 'rel' 的安全路径 '/server'，
    除 'main' 分支，并将分支 'main/Rel2.1' 添加到
    安全路径所用到的分支。就前面的示例而言，
    在分支列表将包含 'main/Rel2.1' 和 'main/2.0'。）

    cm ^acl --^user=vsanchezm -^allowed=-^read -^overrideallowed=+^read ^path:/doc
    （除授 'vsanchezm' 的 '^read' 限，在 '/doc' 路径中其行覆盖。）

== CMD_DESCRIPTION_ADD ==
向版本控制中添加。

== CMD_USAGE_ADD ==
用法：

    cm ^add [-^R | -^r | --^recursive] [--^silent] [--^ignorefailed]
           [--^skipcontentcheck] [--^coparent] [--^filetypes=<文件>] [--^noinfo]
           [--^format=<格式字符串>] [--^errorformat=<格式字符串>]
           <路径>[ ...]

    路径   要移的一个或多个。使用双引号 (" ") 指定
                包含空格的路径。使用空格各个行分隔。
                使用 * 添加当前目的所有内容。

：

    -^R -^r --^recursive   以方式添加。
    --^silent            不示任何出。
    --^ignorefailed      如果无法添加某一，添加操作将在没有
                        一的情况下行。注意：如果无法添加
                        某个目，不会添加目中的内容。
    --^skipcontentcheck  如果不足以根据展名将文件置
                        文本或二制，会将文件置二制，
                        而不是内容来型。做
                        是了在行大量入提高性能。
    --^coparent          要添加的的父行出。
    --^filetypes         要使用的 filetypes 文件。看以下接以了解
                        更多信息：
                        http://blog.plasticscm.com/2008/03/custom-file-types.html
    --^noinfo            不打印度信息。
    --^format            索特定格式的出消息。看
                        示例以了解更多信息。
    --^errorformat       索特定格式的消息
                        （如果有）。看示例以了解更多信息。

== CMD_HELP_ADD ==
注：

    添加的要求：
    - 要添加的的父目必已事先添加。

从 stdin 取入：

    '^add' 命令可从 stdin 取路径。此，一个破折号
    "-"。
    示例：cm ^add -

    路径将一直取到有空行入止。
    情况下允使用指定要添加的文件。
    示例：
      dir /S /B *.c | cm ^add -
      （在 Windows 中，添加工作区中的所有 .c 文件。）



示例：

    cm ^add file1.txt file2.txt
    （添加 'file1.txt' 和 'file2.txt' 。）

    cm ^add c:\workspace\file.txt
    （添加路径 'c:\workspace' 中的 'file.txt' 。）

    cm ^add -^R c:\workspace\src
    （以方式添加 'src'。）

    cm ^add -^R *
    （以方式添加当前目的所有内容。）

    cm ^add -^R * --^filetypes=filetypes.conf
    （以方式添加当前目的所有内容，使用
    'filetypes.conf' 根据文件的展名文件分配型，
    而不是文件内容。）

    cm ^add --^coparent c:\workspace\dir\file.txt
    （将 'file.txt' 添加到源代管理中，并行 'dir' 的出。）

    cm ^add -^R * --^format="ADD {0}" --^errorformat="ERR {0}"
    （以方式添加当前目的所有内容，
    于成功添加的文件，打印 '^ADD <>'，而于无法添加的，
    打印 '^ERR <>'。）

== CMD_DESCRIPTION_ANNOTATE ==
示文件一行上次修改所在的更集及其作者。

== CMD_USAGE_ANNOTATE ==
用法：

    cm ^annotate | ^blame <格>[ ...]
        [--^format=<格式字符串>]
        [--^ignore=(^eol | ^whitespaces | ^"eol&whitespaces" | ^none)]
        [--^dateformat=<日期格式字符串>]
        [--^encoding=<名称>]
        [--^stats]
        [--^repository=<存格>]

    格        要批注的文件的格。
                （使用 'cm ^help ^objectspec' 可一了解格。）
                使用双引号 (" ") 指定包含空格的路径。

：

    --^format        索特定格式的出消息。参
                    “注”以了解更多信息。
    --^ignore        置指定的比方法。
                    参“注”以了解更多信息。
    --^dateformat    置出格式以打印日期。
    --^encoding      指定出，如：utf-8。
                    参位于以下网址的 MSDN 文档：
                    http://msdn.microsoft.com/en-us/library/system.text.encoding.aspx
                    以看包含受支持及其格式的表格
                    （位于面末尾的“名称”列中）。
    --^stats         示信息。
    --^repository    指定用于算批注的存
                    格。默情况下，此命令使用的
                    存存了工作区中已加的
                    修存。（使用 'cm ^help ^objectspec' 可一
                    了解存格。）

== CMD_HELP_ANNOTATE ==
注：

    无法批注二制文件。

    --^ignore ：
        ^none                行尾和空格差。
        ^eol                 忽略行尾差。
        ^whitespaces         忽略空格差。
        ^"eol&whitespaces"   忽略行尾和空格差。

    --^format ：
        此命令的出参数如下：
        {^owner}        上次更改行的用。
        {^rev}          行的源修格。
        {^content}      行内容。
        {^date}         入行的日期。
        {^comment}      行的源修的注。
        {^changeset}    行的源修的更集。
        {^line}         文件中的行号。
        {^id}            ID。
        {^parentid}     的父 ID。
        {^rep}          的存。
        {^branch}       行的源修的分支。
        {^ismergerev}   是否在合并中建了行的修。

    --^dateformat:
        指定打印日期的出格式。
        参以下网址中指明的受支持格式：
        https://docs.microsoft.com/en-us/dotnet/standard/base-types/custom-date-and-time-format-strings

    --^repository:
        从程存索数据。合用于分布式
        景。

示例：

    cm ^blame c:\workspace\src --^ignore=^"eol&whitespaces" --^encoding=utf-8
    cm ^annotate c:\workspace\file.txt --^ignore=^eol

    cm ^annotate c:\workspace\file.txt --^format="{^owner} {^date, 10} {^content}"
    （依次写入 owner 字段、一个空格、date 字段（右）、
    一个空格以及 content 字段。）

    cm ^blame c:\workspace\file.txt --^format="{^owner, -7} {^comment} {^date}" \
        --^dateformat=yyyyMMdd
    （依次写入 owner 字段（占用 7 个空格，左）、
    一个空格、comment 字段、一个空格以及尾的
    格式化日期（例如，20170329）。）

    cm ^annotate c:\workspace\file.txt --^repository=centralRep@myserver:8084

    cm ^blame ^serverpath:/src/client/checkin/Checkin.cs#^cs:73666
    （使用服器路径更集 73666 中始的文件行批注。）

== CMD_DESCRIPTION_ARCHIVE ==
将数据存档在外部存中。

== CMD_USAGE_ARCHIVE ==
用法：

    cm ^archive | ^arch <修格>[ ...][-^c | --^comment=<注字符串>]
                        [--^file=<基文件>]
    （从存中提取数据并将些数据存在外部存中。）

    cm ^archive | ^arch <修格>[ ...]--^restore
    （将以前存档的修原到存中。）

    修格             一个或多个修格。可通 "-" 修符从 STDIN
                        行取。（使用 'cm ^help ^objectspec' 可
                        一了解修格。）
    --^restore           从生成的存档文件中原以前存档的
                        数据。

：

    -^c | --^comment      在要建的存档存文件中置注。
    --^file              新存档数据文件的名称前和
                        （可）路径。

== CMD_HELP_ARCHIVE ==
注：

    此命令从存数据中提取数据并将些数据存在
    外部存中，从而省数据空。
    此命令可以将以前存档的修原到
    存数据中 (--^restore)。

    使用 'cm ^help ^objectspec' 可了解如何指定修格。

    行此命令的用必是 Unity VCS 服器管理
    （存服器所有者）才能完成操作。

    来自指定修中的个数据段都将存在
    不同的文件中，且文件名以 --^file
    参数定的。此参数可以包含完整路径，其中包括
    未来存档文件的前，也可以包含此前。

    存档后可以通方式来自指定修中的
    数据：

    - 从客端：客端将数据是否已存档，
      并提示用入文件的位置。
      用可以通建一个名 externaldata.conf 的
      文件（在准配置文件位置，使用用于
      client.conf 文件的相同）来配置外部数据位置，
      其中包含已存档数据所在的路径。

    - 从服器：通方式，用将不必知道数据
      是否已存档，因求将以透明的方式由服器
      解析。此，管理将在服器目中建一个
      名 externaldata.conf 的文件，并在文件中填充
      已存档的卷所在的路径。

    要取消存档（原）一个修（或一修），必能
    从客端已存档的文件。因此，无法取消存档
    服器正在解析的数据（方法 2），原因是客端
    无法将数据已存档。如果使用了方法 2，
    了成功取消存档，管理必首先
    externaldata.conf 服器文件，以便除必取消存档的
    已存档文件的限。

    置 PLASTICEDITOR 境量可指定用于入注的
    器。

示例：

    cm ^archive bigfile.zip#^br:/main
    （在分支 'main' 中存档 'bigfile.zip' 的最后一个修。）

    cm ^archive ^rev:myfile.pdf#^cs:2 -^c="大型 PDF 文件" --^file=c:\arch_files\arch
    （将更集 2 中 myfile.pdf 的修存档在 'c:\archived_files'
    文件中。存档的文件名将以 'arch' （例如 arch_11_56）。）

    cm ^find "^revs ^where ^size > 26214400" --^format="{^item}#{^branch}" \
      --^nototal | cm ^archive --^comment="volume00" --^file="volume00" -
    （将所有大于 25Mb 的文件存档在名称 'volume00' 的
    文件中。）

    cm ^find "^revs ^where ^size > 26214400 ^and ^archived='true'" \
      --^format="{^item}#{^branch}" --^nototal | cm ^archive --^restore
    （原所有大于 25Mb 的存档文件。）

      相信息。
    - 如果包含 '--^from' ，命令示
      在建指定更集的分支中从更集到最后一个
      更集的个更集的相信息。
    - 如果示不隔的 '--^allbranches'，
      命令会索与指定“更集格”相同的
      信息。
    - 如果使用 '--^from'，出包含从“更集格起点”+1 始的
      更集的信息。
    - 用于示更集信息的存就是在行命令的
      路径中加的存。

    此命令接受的格式字符串 ('--^itemformat') 以及
    更集的格式字符串 ('--^csformat')。

    '--^csformat' 的出参数如下：
        {^tab}           插入一个制表符空格位。
        {^newline}       插入一个新行。
        {^changesetid}   更集号。
        {^branch}        在其中建更集的分支。
        {^date}          更集的日期。
        {^owner}         更集的所有者。
        {^comment}       更集的注。
        {^items}         更集内及的。
        {^repository}    更集所在的存。
        {^repserver}     服器名称。

    '--^itemformat' 的出参数如下：
        {^tab}           插入一个制表符空格位。
        {^newline}       插入一个新行。
        {^path}          路径。
        {^branch}        在其中建更集的分支。
        {^date}          更集的日期。
        {^owner}         更集的所有者。
        {^shortstatus}   打印短格式。参下文。
        {^fullstatus}    打印格式。参下文。

        短格式及其的格式：
            '^A'   ^Added
            '^D'   ^Deleted
            '^M'   ^Moved
            '^C'   ^Changed

    些是有效的出字符串：
        --^csformat="{^newline}更集 {^changesetid} 建于 {^date}；{^tab} 已更改：{^items}。"
        --^itemformat="{^newline}分支 {^branch} 中的 {^path} 已更改。"

示例：

    cm ^log
    （示个分支中上个月建的个更集的相
    信息。）

    cm ^log ^cs:16
    （示在其中建更集的分支内的更集 16 中
    所做更改的相信息。）

    cm ^log ^cs:16 --^csformat="{^newline}更集 {^changesetid} 建于 \
      {^date}；{^tab} 已更改：{^items}。"
    （按指定格式示此信息。）

    cm ^log --^from=^cs:20 ^cs:50
    （示从更集 21 到更集 50 的个更集中包含的
    个修的相信息。）

    cm ^log --^from=^cs:20 ^cs:50 --^allbranches
    （示存的个分支中从更集 21 到
    更集 50 的个更集中包含的个修的
    相信息。）

    cm ^log ^rep:myrep@localhost:8084
    （示在指定存中行的更改的相信息。
    无需工作区也可行此命令。）

    cm ^log --^from=^cs:20@^rep:mainRep@localhost:8084
    （示从更集 21 始的个更集内包含的个修
    的相信息。无需工作区也可行此命令，因
    已指定完整更集格。）

== CMD_DESCRIPTION_MERGE ==
将一个分支与一个分支合并。

== CMD_USAGE_MERGE ==
用法：

    cm ^merge <源格> [--^merge] [--^cherrypicking] [--^forced]
                           [--^mergetype=(^onlyone|^onlysrc|^onlydst|^try|^forced)]
                           [--^interval-origin=<更集格> | --^ancestor=<更集格>]
                           [--^keepsource | --^ks] [--^keepdestination | --^kd]
                           [--^automaticresolution=<冲突型>[;...]]
                           [--^subtractive] [--^mount] [--^printcontributors]
                           [--^noprintoperations] [--^silent]
                           [(--^to=<分支格> | --^destination=<分支格>)[--^shelve]]
                           [--^no-dst-changes]
                           [-^c=<注字符串> | --^commentsfile=<注文件>]
                           [--^resolveconflict --^conflict=<索引>
                           --^resolutionoption=(^src|^dst|(^rename --^resolutioninfo=<字符串名称>))
                           --^mergeresultfile=<路径> --^solvedconflictsfile=<路径>]
                           [--^nointeractiveresolution]
                           [--^machinereadable [--^startlineseparator=<分隔符>]
                             [--^endlineseparator=<分隔符>] [--^fieldseparator=<分隔符>]]

    源格           要合并的源象的格：
                          - 分支格：'[^br:/]br_name'
                          - 格：'^lb:lb_name'
                          - 更集格：'^cs:cs_number'
                          - 置格：'^sh:shelve_number'
                          （使用 'cm ^help ^objectspec' 可一了解格。）

    --^merge                   行合并。否，打印找到的
                              冲突。
    --^cherrypicking           合并源更集内包含的
                              更。如果合并源格是，
                              不会使用此。
    --^forced                  不源和目是否
                              已接。
                              此可用于隔合并
                              和挑。
    --^mergetype               参“注”以了解更多信息。
    --^interval-origin         指定将个更集
                              隔原点，以便合并考
                              源更集与指定隔原点
                              之的差。
    --^ancestor                是 --^interval-origin 的名。
    --^keepsource              于有冲突的，接受源参与者
                              的所有更改。
    --^keepdestination         于有冲突的，保留目参与者
                              的更改。
    --^automaticresolution     用于解决目冲突。此
                              可决定自
                              源参与者是目参与者
                              来解决冲突。
                              使用分号各个冲突型行分隔。
                              参“注”以了解更多信息。
    --^subtractive             除由合并引入的更改。
                              命令的参数（源格）用于
                              指定要除个源的
                              更改。必是更集。如果是更集
                              隔，必使用 '--^interval-origin' 来
                              定隔原点。了除更改，
                              系将建一个新的已出修，
                              修将具有上一个
                              修的内容（不包含已除的更改）。
    --^mount                   定存的装入点。
    --^printcontributors       打印参与者（基、源和
                              目）。
    --^noprintoperations       以静默方式解决合并，
                              不示有解决情况的信息。
    --^silent                  不示任何出。
    --^to | --^destination      指定分支行“合并到”操作
                              （通入分支格）
                              并完全解决冲突。
                              “合并到”（或无工作区的合并）是在
                              服器端行的合并。正常的合并
                              生在“从”分支、或更集合并的
                              工作区上，而“合并到”完全生
                              在服器上。在正常合并中，
                              “目”是工作区，而在“合并到”中，
                              必始指定目（就是称之
                              “到”的原因）。
                              看以下接以了解于
                              “合并到”功能的更多信息：
                              https://www.plasticscm.com/download/help/mergeto
    --^shelve                  使用合并果的更改（加上合并
                              可追溯性信息）来建一个置，
                              而不是建新的更集。合并源
                              置，此不可用。
                              可用于服器端合并
                              （又称“合并到”）。因此，需要 '--^to' 和
                              '--^merge' 。
    --^no-dst-changes          保目参与者不具有
                              更改（目更集也是
                              共同上）。目上存在更改，
                              不允合并。
    -^c                        将指定的注用于合并操作中
                              建的更集。
    --^commentsfile            将指定文件中的注用于合并操作中
                              建的更集。
    --^resolveconflict         （主要由插件使用。参“注”以了解更多信息。）
                              用于解决目冲突。
    --^conflict                与 '--^resolveconflict' 志合使用，指定
                              要解决的冲突的索引（从 1 始）。
    --^resolutionoption        与 '--^resolveconflict' 志合使用，指示
                              冲突解决型。使用以下
                              之一：'^src'、'^dst'、'^rename'。
                              参“注”以了解更多信息。
    --^resolutioninfo          与 '--^resolveconflict' 志合使用，提供在
                              '--^resolutionoption'  'rename' 
                              要使用的名称。
    --^mergeresultfile         与 '--^resolveconflict' 志合使用，将
                              不同用之的合并果的信息
                              出到文件中。指定的路径将在第一次
                              用期建，并在个后
                              用更新。
    --^solvedconflictsfile     与 '--^resolveconflict' 志合使用，
                              将不同用之已解决的冲突
                              的信息出到文件中。指定的路径
                              将在第一次用期建，并在个
                              后用更新。
    --^nointeractiveresolution （主要由插件使用。参“注”以了解更多信息。）
                              避免提示用已生手冲突。
                              就不会解决目冲突。
    --^machinereadable         （主要由插件使用。参“注”以了解更多信息。）
                              以易于解析的格式出果。
    --^startlineseparator      与 '--^machinereadable' 志合使用，指定
                              行如何。（默：空字符串。）
    --^endlineseparator        与 '--^machinereadable' 志合使用，指定
                              行如何尾。（默：空字符串。）
    --^fieldseparator          与 '--^machinereadable' 志合使用，指定
                              如何分隔字段。（默：
                              空格。）

== CMD_HELP_MERGE ==
注：

    此命令用于合并个分支之或与
    分支之的更改。合并的目必始分支。
    合并源将指定参数。
    目是工作区的当前内容。
    例如，要示从分支 task001 合并到主分支
    的元素，器必指向主分支，
    工作区必完成更新，然后：
        cm ^merge ^br:/task001

    添加 '--^merge' 来切行合并：
        cm ^merge ^br:/task001 --^merge

    要定合并源，可以使用以下格：

    - 分支格：
        [^br:/]br_name
        示例：^br:/main/task001
        （以上示例从此分支上的最后一个更集行合并。）

    - 格：
        ^lb:lb_name
        示例：^lb:BL001
        （从的更集合并。）

    - 更集格：
        ^cs:cs_number
        示例：^cs:25
        （从定的更集内容合并。）

    - 置格：
        ^sh:shelve_number
        示例：^sh:2
        （从定的置内容合并。）

    要自解决目冲突，使用 '--^automaticresolution'
    并指定冲突型，然后指定在合并操作
    期必的参与者（源或目）。
    （使用分号 (;) 各个“冲突型”-“参与者”行分隔。）
    例如：
        cm ^merge ^cs:2634 --^merge --^automaticresolution=^eviltwin-src;^changedelete-src
        （自更集 2634 始的合并操作将会解决 "^eviltwin" 和
        "^changedelete" 冲突，解决方法是在情况下保留源 ("-^src")
        参与者）。
    - 冲突型之后的 "-^src" 后告合并命令保留
      源参与者更改。
    - "-^dst" 后将保留目参与者更改。
    以下是合并命令支持的冲突型的列表：
      "^movedeviltwin"、"^eviltwin"、"^changedelete"、"^deletechange"、"^movedelete"、
      "^deletemove"、"^loadedtwice"、"^addmove"、"^moveadd"、"^divergentmove"、
      "^cyclemove"、"^all"。
    "^all" 会覆盖其他。在以下示例中，
    "^eviltwin-dst" 将被忽略：
        cm ^merge ^br:/main/task062 --^merge --^automaticresolution=^all-src;^eviltwin-dst
    看以下接以了解于合并冲突的更多信息：
    https://www.plasticscm.com/download/help/directorymerges

    以下是 '--^mergetype' 的：
        ^onlyone         有一个参与者修改了的情况下，才行
                        自合并。
        ^onlysrc         有一个源参与者修改了的情况下，才行
                        自合并。
        ^onlydst         有一个目参与者修改了的情况下，才行
                        自合并。
        ^try             有一个参与者修改了冲突代段（个冲突）
                        的情况下，才行自合并。
        ^forced          始解决所有非自冲突。

    以下是主要由插件和集成使用的：
        - '--^resolveconflict'，用于解决目冲突。必使用
          以下：
              - '--^conflict' 是要解决的冲突的索引
                （从 1 始）。
              - '--^resolutionoption' 指示要使用的冲突解决
                 方法。可以是：
                    - '^src'，用于保留源更改而放弃
                      目更改
                    - '^dst'，用于保留目更改而放弃
                      源更改
                    - '^rename'（在冲突型支持此解决方法
                      的情况下），用于将目重命名
                      通 '--^resolutioninfo' 提供的定名称。
                        - '--^resolutioninfo'，用于提供在 '^rename' 解决
                          方法中使用的名称
              - '--^mergeresultfile' 和 '--^solvedconflictsfile'，者用于
                存不同用之的合并信息。
        - '--^nointeractiveresolution' 指示合并操作不要求用
          行手冲突解决。
        - '--^machinereadable' 和 '--^startlineseparator'、'--^endlineseparator'、
          '--^fieldseparator' ，用于以机器可的方式
          （易于解析）打印出。
        示例：
        cm ^merge --^machinereadable --^startlineseparator=start@_@line \
          --^endlineseparator=new@_@line --^fieldseparator=def#_#sep \
          --^mergeresultfile=C:\Users\Borja\AppData\Local\Temp\2tmp4D6C.tmp \
          --^solvedconflictsfile=C:\Users\Borja\AppData\Local\Temp\2tmp4D6D.tmp \
          --^resolveconflict --^conflict=1 --^resolutionoption=rename  \
          --^resolutioninfo=bin_dst ^br:/main/task --^merge

示例：

    cm ^merge ^br:/task001
    （不合并，打印要合并的。）

    cm ^merge ^br:/task001 --^merge
    （从分支 'task001' 行合并。）

    cm ^merge ^cs:5 --^merge --^cherrypicking --^interval-origin=^cs:2
    （从更集隔 (2,5] 行挑。）

    cm ^merge ^cs:8 --^merge --^subtractive --^keepdestination
    （从更集 8 行法合并，保留那些有冲突的
    元素的目更改。）

    cm ^merge ^br:/main/task001 --^to=^br:/main --^merge -^c="集成的新 UI"
    （行从分支 'task001' 到分支 'main' 的服器端合并
    （又称“合并到”），并置注。）

    cm ^merge ^br:/main/task001 --^to=^br:/main --^merge --^shelve
    （行从分支 'task001' 到分支 'main' 的服器端合并，
    并将果保留在置中。）

    cm ^merge ^sh:2 --^to=^br:/main --^merge --^no-dst-changes
    （在从当前的 'main' 部建，才将置 2
    用于 'main'）

== CMD_DESCRIPTION_BRANCH_CREATE ==
建新的分支。

== CMD_USAGE_BRANCH_CREATE ==
用法：

    cm ^branch | ^br [^create | ^mk] <分支格>
                   [--^changeset=<更集格> | --^label=<格>]
                   [-^c=<注字符串> | -^commentsfile=<注文件>]

    分支格         新分支名称或格。
                   （使用 'cm ^help ^objectspec' 可一了解分支格。）

：

    --^changeset     用作新分支的起点的更集。
                    （使用 'cm ^help ^objectspec' 可一了解更集格。）
    --^label         用作新分支的起点的。
                    （使用 'cm ^help ^objectspec' 可一了解格。）
    -^c              用指定的文本填充新分支的“注”
                    字段。
    -^commentsfile   用指定文件的内容填充新分支的“注”
                    字段。

== CMD_HELP_BRANCH_CREATE ==
注：

    要建分支，指定不含任何次的名称。
    例如：

        cm ^br /dev

    如果未指定可参数 '--^changeset'，新分支的基
    将是父分支上的最后一个更集。如果新分支
    是分支，使用的基更集将更集 0。

    可以使用 '-^c' 或 '-^m' 来指定注：

        cm ^branch /main/task001 -^c="是注"
        cm ^branch /main/task001 -^m "是注"

    置 PLASTICEDITOR 境量可指定用于入注的
    器。如果已置 PLASTICEDITOR 境量，
    并且注空，会自器
    以用于指定注。

示例：

    cm ^branch task001
    cm ^branch ^create task001
    cm ^branch ^mk task001
    cm ^br ^mk task001
    （在当前工作区的存中建分支
    'task001'。）

    cm ^branch ^br:/task001/task002@
    （建 'task002' 分支作 'task001' 的子。）

    cm ^br /main/task001@myrep@myserver:8084 -^c="我的注"
    （在存 'myrep@myserver:8084' 中，建 'task001' 分支
    作 'main' 的子，并包含注 '我的注'。）

    cm ^branch ^br:/main/task001 --^changeset=2837 -^commentsfile=commenttask001.txt
    （建 'task001' 分支作 'main' 的子，基更集 'changeset=2837'，
    并用 'commenttask001.txt' 文件中的注。）

== CMD_DESCRIPTION_OBJECTSPEC ==
描述如何写象格。

== CMD_USAGE_OBJECTSPEC ==
用法：
    cm ^objectspec
    取于如何建象格的所有信息。

== CMD_HELP_OBJECTSPEC ==
一些 Unity VCS 命令要求将 '象格' 作入来引用
定象（通常是分支、更集、存等）。

本文档介可用的不同“格”以及如何
建格。

个格型以唯一（例如 "^rep:" 或 "^cs:"）。必
采用一般象格的命令指定（例如：
"cm ^setowner 象格" 命令），但于采用一型格的命令
常常可以省略（例如："cm ^getfile 修格" 命令）。

-- 存服器格 --
    ^repserver:name:port

    示例：
        cm ^repo ^list ^repserver:skull:8084
        cm ^repo ^list skull:8084

    旁注：
        由于史原因，我称之“存服器格”，而不只是
        “服器格”。很久以前，我有独的工作区服器和
        存服器，命名方式沿用至今。

-- 存格 --
    ^rep:rep_name@[存服器格]

    示例：
        cm ^showowner ^rep:codice@localhost:6060
        （此的 "^rep:" 是必需的，因 ^showowner 不容存，
        而且容其他型的象。因此，需要用指明
         象型。）

-- 分支格 --
    ^br:[/]br_name[@repspec]

    示例：
        cm ^switch ^br:/main@^rep:plastic@^repserver:skull:9095
        （情况下不需要 "^br:"、"^rep" 和 "^repserver"，所以
         命令容短很多的格式：
        "cm ^switch main@plastic@skull:9095"。）

        cm ^find ^revisions "^where ^branch='^br:/main/task001'"

    注：
        分支上的初始 '/' 不是必需的。我曾将所有分支
        指定 /main、/main/task001，依此推。但是在，我首
        更短的 main、main/task001 形式，使命令更。

-- 更集格 --
    ^cs:cs_number|cs_GUID[@repspec]

    可以指定更集的号或 GUID。

    示例：
        cm ^ls /code --^tree=ae1390ed-7ce9-4ec3-a155-e5a61de0dc77@code@skull:7070

-- 格 --
    ^lb:lb_name[@repspec]

    示例：
        cm ^switch ^lb:RELEASE2.0
        cm ^switch ^lb:RELEASE1.4@myrep@MYSERVER:8084

-- 修格 --
有不同型的修格：

    ^rev:item_path[#(分支格|更集格|格)]

    ^rev:^serverpath:item_path#(分支格|更集格|格)

    ^rev:^revid:rev_id[@rep_spec]

    ^rev:^itemid:item_id#(分支格|更集格|格)

    示例：
        cm ^diff ^rev:readme.txt#^cs:19 ^rev:readme.txt#^cs:20

        cm ^diff ^serverpath:/doc/readme.txt#^cs:19@myrepo \
            ^serverpath:/doc/readme.txt#^br:/main@myrepo@localhost:8084

        cm ^cat ^revid:1230@^rep:myrep@^repserver:myserver:8084

-- 格 --
    ^item:path
    少使用。

    示例：
        cm ^find ^revision "^where ^item='^item:.'"

-- 属性格 --
    ^att:att_name[@repspec]

    示例：
        cm ^attribute ^set ^att:merged@code@doe:8084 ^cs:25@code@doe:8084 完成

-- 置格 --
    ^sh:sh_number[@repspec]

    示例：
        cm ^diff ^sh:2 ^sh:4

-- 工作区格 --
    ^wk:name@clientmachine

少使用，因用于工作区相的命令。合用于
按名称和机器（而不是路径）来指定工作区。

    示例：
        cm ^showselector ^wk:codebase@modok

    旁注：
        些格来自早期的 Plastic SCM 2.x，当的
        “工作区服器”是一集中存工作区元数据
        的方式。后期由于性能而被弃用。

== CMD_DESCRIPTION_PARTIAL_UNDO ==
撤工作区中的更改。

== CMD_USAGE_PARTIAL_UNDO ==
用法：

    cm ^partial ^undo [<路径>[ ...]] [--^symlink] [-^r | --^recursive]
                    [<器>[ ...]]
                    [--^silent | --^machinereadable [--^startlineseparator=<分隔符>]
                                [--^endlineseparator=<分隔符>] [--^fieldseparator=<分隔符>]]

    路径                    要用操作的文件或目
                            的路径。使用双引号 (" ") 指定
                            包含空格的路径。使用空格各个路径
                            行分隔。
                            如果未指定路径，默情况下，
                            撤操作将用于当前目中的所有
                            文件。
    器                  将一个或多个指定器用于定
                            路径。使用空格各个器行分隔。参
                            “器”部分以了解更多信息。

：

    --^symlink               将撤操作用于符号接而不是
                            目。
    -^r                      以方式行撤。
    --^silent                不示任何出。
    --^machinereadable       以易于解析的格式出果。
    --^startlineseparator    与 '--^machinereadable' 志合使用，指定
                            行如何。
    --^endlineseparator      与 '--^machinereadable' 志合使用，指定
                            行如何尾。
    --^fieldseparator        与 '--^machinereadable' 志合使用，指定
                            如何分隔字段。

器：

    如果未指定志，默情况下将撤所有更改，但是
    可以使用以下一个或多个志来路径。
    如果文件或目匹配一个或多个指定的更改，
    将撤文件或目上的所有更改。
    例如，如果同指定了 '--^checkedout' 和 '--^moved'，在
    已出并移文件的情况下，更改都将被撤。

    --^checkedout            已出的文件和目。
    --^unchanged             内容不的文件。
    --^changed               本地更改的或出的文件和
                            目。
    --^deleted               已除的文件和目。
    --^moved                 已移的文件和目。
    --^added                 已添加的文件和目。

== CMD_HELP_PARTIAL_UNDO ==
注：

    ^undo 命令有，因命令在撤工作后将不可逆。
    ^undo 完成后，无法将受其影的文件和目
    恢到先前状。如果参数中
    未指定路径，默情况下，命令将撤
    当前目中的所有更改，但不采用方式。
    从 /src 目行，以下命令是等效的：

        /src
        |- file.txt
        |- code.cs
        \- /test
           |- test_a.py
           \- test_b.py

        cm ^partial ^undo
        cm ^partial ^undo *
        cm ^partial ^undo file.txt code.cs /test

        cm ^partial ^undo .
        cm ^partial ^undo /src file.txt code.cs

    如果希望操作以方式行，必指定 '-^r' 志。

    撤某一目下的所有更改（包括影目
    本身的更改）：

        cm ^partial ^undo 目路径 -^r

    如果目路径是工作区路径，工作区中的个更改都将
    被撤。

示例：

    cm ^partial ^undo .-^r
    （以方式撤当前目中的所有更改。如果从工作区的
    根目行，撤整个工作区中的所有更改。）

    cm ^partial ^co file.txt
    cm ^partial ^undo file.txt
    （撤 file.txt 的出。）

    ^echo ^content >> file.txt
    cm ^partial ^undo file.txt
    （撤 file.txt 的本地更改。）

    cm ^partial ^undo src
    （撤 src 目及其文件的更改。）

    cm ^partial ^undo src/*
    （撤 src 中包含的个文件和目的更改，而不会
    影 src。）

    cm ^partial ^undo *.cs
    （撤当前目中与 *.cs 匹配的个文件或目的
    更改。）

    cm ^partial ^undo *.cs-^r
    （撤当前目以及下个目中与 *.cs 匹配的
    个文件或目的更改。）

    cm ^partial ^co file1.txt file2.txt
    ^echo ^content >> file1.txt
    cm ^partial ^undo --^unchanged
    （撤未更改的 file2.txt 的出，并忽略本地更改的
    file1.txt。）

    ^echo ^content >> file1.txt
    ^echo ^content >> file2.txt
    cm ^partial ^co file1.txt
    cm ^partial ^undo --^checkedout
    （撤已出文件 file1.txt 中的更改，并忽略 file2.txt，
    因后者未出。）

    cm ^partial ^add file.txt
    cm ^partial ^undo file.txt
    （撤添加 file.txt，使文件再次成私有文件。）

    ^rm file1.txt
    ^echo ^content >> file2.txt
    cm ^partial ^add file3.txt
    cm ^partial ^undo --^deleted --^added *
    （撤 file1.txt 的除和 file3.txt 的添加，并忽略 file2.txt
    的更改。）

== CMD_HELP_PULL ==
注：

    '^pull' 命令可以在源存与目存之
    制分支（以及些分支的更集）。
    些存可以位于不同服器中。

    有制操作：'^push' 和 '^pull'。

    '^pull' 操作表示制操作要求将数据从源存
    存到目存。
    客端将接到目存，然后建立
    从主机到源存的接以便索
    目数据。在拉取期，目服器将
    接到源。

    尽管在典型的分布式景中，者会将数据
    从本地服器推送到主服器，但者可能也希望
    从主服器拉取最新的存更新。

    制操作可以解决在同一分支上个制的存
    已行并更改的情况：

    - 推送：如果将数据推送到一个存，但个存中的
      更改比送的更改更新，系会要求提取最新的更改，
      解决合并操作，最后再次推送。

    - 拉取：次从程分支拉取更集，都会将些更集
      正接到相的父更集。如果拉取的更集不是
      分支中最后一个更集的子，会出
      一多景。分支将具有多个“部”，即分支中的
      最后一个更集。此需要先合并个“部”，
      然后才能再次推送。

    可以按模式行拉取：

    1) 服器之的直接通信：目服器将
       从源服器提取数据，自指定
       分支同数据。

    2) 使用推送和 '--^package' 来入先前生成的包。

    模式 1) 要求程服器行命令的用行身，
    方法是使用 client.conf 文件中的默身，
    或者指定 '--^authmode' 和 '--^authdata' 修符
    或 '--^authmode'、'--^user' 和 '--^password'（如果身模式
    ^UPWorkingMode）。

    模式 2) 要求使用先前通推送命令生成的
    包文件。

    住，拉取制是以接方式行的。行，
    命令要求目存接到源
    并取所分支。

    然而，可以使用推送命令来直接一点。
    因此，命令会将所分支从源制到
    目。

示例：

    cm ^pull ^br:/main@project1@remoteserver:8084 projectx@myserver:8084
    （将 'main' 分支从 'remoteserver' 拉取到 'myserver'。情况下，

      --^authmode=^LDAPWorkingMode --^authdata=::0:dave:fPBea2rPsQaagEW3pKNveA
    （拉取与以前相同的分支，但在将程服器配置
    使用 Active Directory 用行身。例如，我正在从 Linux
    算机接到配置使用 Active Directory 集成模式
    的 Windows 服器。我会指定自己的 Active Directory 用名和密，
    并以 LDAP 形式将此信息服器。）

    cm ^pull ^br:/main@project1@remoteserver:8084 projectx@myserver:8084 \
      --^authmode=^UPWorkingMode --^user=dave --^password=mysecret
    （拉取同一个分支，但在利用 Unity VCS 中
    包含的用名/密数据在程服器上用
    行身。）

    cm ^pull ^br:/main@project1@remoteserver:8084 projectx@myserver:8084 --^nodata
    （将 'main' 分支从 'remoteserver' 制到 'myserver' 而不制数据。）

    cm ^pull ^hydrate ^br:/main@projectx@myserver:8084 projectx@remoteserver:8084
    （从程服器取数据， 'main' 分支中的所有更集
    行 hydrate 操作。）

    cm ^pull ^hydrate ^cs:122169@projectx@myserver:8084 projectx@remoteserver:8084

                        如果未指定，命令将使用用的
                        默存服器。

                        而不是存。
                        合在没有直接网接用于在服器

        具有“新名称”的用。

    1) 使用身参数：--^authmode=<模式> --^authdata=<数据>

        中的 <^SecurityConfig> 条目的内容。可以从
        Unity VCS GUI 中的“首”下的“制配置文件”
        松生成 profiles.conf 文件。

        如果使用 ^UPWorkingMode，只需指定：

        --^authmode=^UPWorkingMode --^user=<用> --^password=<密>

    2) 身文件，于接到的个服器，文件可能会
       有所不同，其中包含服器的凭据。

        --^authfile=<身文件>
            文件包含 2 行：
            第 1 行是模式，如 '--^authmode' 中所述
            第 2 行是身数据，如 '--^authdata' 中所述

    cm ^shelveset ^apply <置格> [--^mount] [--^encoding=<名称>]
                       [--^comparisonmethod=(^ignoreeol | ^ignorewhitespaces| \
                                            ^ignoreeolwhitespaces | ^notignore)]

    置格             置集格。（使用 'cm ^help ^objectspec' 可
                        一了解置集格。）

：

    --^mount             定存的装入点。
    --^encoding          指定出，如：utf-8。
                        参位于以下网址的 MSDN 文档：
                        http://msdn.microsoft.com/en-us/library/system.text.encoding.aspx
                        以看包含受支持及其格式的表格
                        （位于面末尾的“名称”列中）。
    --^comparisonmethod  置比方法。参“注”以了解更多信息。

== CMD_HELP_SHELVESET_APPLY ==
'^shelveset ^apply' 命令可原存的置集的内容。

注：

    比方法：
        ^ignoreeol               忽略行尾差。
        ^ignorewhitespaces       忽略空格差。
        ^ignoreeolwhitespaces    忽略行尾和空格差。
        ^notignore               行尾和空格差。

示例：

    cm ^shelveset ^apply ^sh:3
    （用存的置。）

与制相的字段：
    多象会跟踪制数据，意味着 Unity VCS 会跟踪最初建制
    的位置。

    可以使用的字段包括：

        ^ReplSrcServer     ：存格。表示“制源服器”。
                            从中制象的服器。

              示例：
                            cm ^find ^branch "^where ^replsrcserver='skull.codicefactory.com:9095'"
                            （找从服器 'skull' 制的分支。）

        ^ReplSrcRepository ：字符串。表示“制源存”。是
                            从中制象的存。

              示例：
                            cm ^find ^branch "^where ^replsrcserver = 'skull.codicefactory.com:9095' \
                              ^and ^replsrcrepository = 'codice'"
                            （找从服器 'skull' 和存 'codice'
                            制的分支。）

        ^ReplLogId         ：整数。制操作的 ID。在 Unity VCS 中，
                            次从本建新象，
                            都会建一个新的 'replicationlog'。

              示例：
                            cm ^find ^revision "^where ^repllogid = 2019974"
                            （找从本 2019974 制
                            的修。）

        ^ReplSrcDate       ：日期。是生制的
                            日期。
                            制的象将保留其原始建
                            日期，如果要找在特定
                            范内制的象，此字段
                            很有用。

              示例：
                            cm ^find ^label "^where ^replsrcdate >= '^one ^month ^ago' \
                              ^and ^date >= '15 ^days ^ago'"
                            （找 15 天前建并在一个月前
                            制的。）
                     
                            cm ^find ^replicationlog "^where ^date > '^one ^week ^ago'"
                            8780433  27/09/2018 8:49:38 codice@BACKYARD:8087 F   mbarriosc
                            （找一周前建的制日志。）

                            在可以制的分支是否是
                            在制之前建的：

                            cm ^find ^branch "^where ^repllogid = 8780433"
                            8780443  26/09/2018 12:20:55 /main/scm23078 maria    codice T

        ^ReplSrcId         ：整数。是制源服器的 ID。
                            可以使用 'cm ^find' 命令搜索
                            '^replicationsource' 象来此 ID。

              示例：
                            cm ^find ^replicationsource
                            7860739  codice@AFRODITA:8087 d9c4372a-dc55-4fdc-ad3d-baeb2e975f27
                            8175854  codice@BACKYARD:8087 66700d3a-036b-4b9a-a26f-adfc336b14f9

                            在可以找从 codice@AFRODITA:8087
                            制的更集：

                            cm ^find ^changesets "^where ^replsrcid = 7860739"


日期常量：
    可以使用遵循算机本地化置的日期格式。
    例如，如果算机以 'MM-dd-yyyy' 格式示日期，
    可以在中使用 '12-31-2019' 之的日期。

    可以使用以下常量来化：
        '^today'         ：今天的日期。
        '^yesterday'     ：昨天的日期。
        '^this ^week'     ：本周星期一的日期。
        '^this ^month'    ：本月第一天的日期。
        '^this ^year'     ：本年 1 月 1 日。
        '^one ^day ^ago'   ：当前日期前一天。
        '^one ^week ^ago'  ：当前日期前七天。
        '^one ^month ^ago' ：当前日期前一个月。
        'n ^days ^ago'    ：当前日期前 'n' 天。
        'n ^months ^ago'  ：当前日期前 'n' 个月。
        'n ^years ^ago'   ：当前日期前 'n' 年。

    以下 '^where' 子句于型 '^date' 的字段有效：
        '(...)^where ^date > '^today' (...)'
        '(...)^where ^date < '^yesterday' (...)'
        '(...)^where ^date > '^this ^week' (...)'
        '(...)^where ^date > '^this ^month' (...)'
        '(...)^where ^date < '^one ^day ^ago' ^and ^date > '3 ^days ^ago' (...)'
        '(...)^where ^date < '^one ^week ^ago' ^and ^date > '3 ^weeks ^ago' (...)'
        '(...)^where ^date < '^one ^month ^ago' ^and ^date > '3 ^months ^ago' (...)'
        '(...)^where ^date > '1 ^year ^ago' (...)'

    可以在 'cm ^find' 命令中制使用特定日期格式，方法是使用
    --^dateformat 志。看 'cm ^find --^help' 以了解更多信息。

== CMD_USAGE_UNDO ==

== CMD_HELP_UNDO ==
注：

    '^undo' 命令有，因命令在撤工作后将不可逆。
    ^undo 完成后，无法将受其影的文件和目
    恢到先前状。如果参数中
    未指定路径，默情况下，命令将撤
    当前目中的所有更改，但不采用方式。
    从 /src 目行，以下命令是等效的：

        /src
        |- file.txt
        |- code.cs
        \- /test
           |- test_a.py
           \- test_b.py

        cm ^undo
        cm ^undo *
        cm ^undo file.txt code.cs /test

        cm ^undo .
        cm ^undo /src file.txt code.cs

    如果希望操作以方式行，必指定 '-^r' 志。

    撤某一目下的所有更改（包括影目
    本身的更改）：

        cm ^undo 目路径 -^r

    如果目路径是工作区路径，工作区中的个更改都将
    被撤。

已除：

    要撤文件和目的除，必指定的
    完整路径，或者指定所属目并使用 ('-^r')
    志。

    例如：

    cm ^undo .
    （不撤当前目中的除。）

    cm ^undo .-^r
    （以方式撤当前目中的所有除（和其他更改）。）

    cm ^undo src/file.txt
    （撤 src/file.txt 的除（或其他更改）。）

示例：

    cm ^undo .-^r
    （以方式撤当前目中的所有更改。如果从工作区的
    根目行，撤整个工作区中的所有更改。）

    cm ^co file.txt
    cm ^undo file.txt
    （撤 'file.txt' 的出。）

    ^echo ^content >> file.txt
    cm ^undo file.txt
    （撤 'file.txt' 的本地更改。）

    cm ^undo src
    （撤 src 目及其文件的更改。）

    cm ^undo src/*
    （撤 src 中包含的个文件和目的更改，而不会
    影 src。）

    cm ^undo *.cs
    （撤当前目中与 *.cs 匹配的个文件或目的
    更改。）

    cm ^undo *.cs -^r
    （撤当前目中以及下所有目中与 *.cs 匹配的
    个文件或目的更改。）

    cm ^co file1.txt file2.txt
    ^echo ^content >> file1.txt
    cm ^undo --^unchanged
    （撤未更改的 'file2.txt' 的出，并忽略本地更改的
    'file1.txt'。）

    ^echo ^content >> file1.txt
    ^echo ^content >> file2.txt
    cm ^co file1.txt
    cm ^undo --^checkedout
    （撤已出文件 'file1.txt' 中的更改，并忽略 'file2.txt'，
    因后者未出。）

    cm ^add file.txt
    cm ^undo file.txt
    （撤添加 'file.txt'，使文件再次成私有文件。）

    ^rm file1.txt
    ^echo ^content >> file2.txt
    cm ^add file3.txt
    cm ^undo --^deleted --^added *
    （撤 'file1.txt' 的除和 'file3.txt' 的添加，并忽略 'file2.txt'
    的更改。）

== CMD_DESCRIPTION_XLINK ==
建、或示 Xlink 的信息。

== CMD_USAGE_XLINK ==
用法：

    cm ^xlink [-^w] [-^rs] <Xlink_路径> / (<更集格> | <格> | <分支格)>
             [<展>[ ...]]
    （建一个 Xlink。）

    cm ^xlink [-^rs] <Xlink_路径> /<相路径> (<更集格> | <格> | <分支格>)
             [<展>[ ...]]
    （建指向 /<相路径>（而不是指向默根目 /）的只
    部分 Xlink。）

    cm ^xlink -^e <Xlink_路径> (<更集格> | <格> | <分支格>)
    （ Xlink 以更改目格。）

    cm ^xlink -^s|--^show <Xlink_路径>
    （示 Xlink 信息，包括展。）

    cm ^xlink -^ar|--^addrules <Xlink_路径> <展>[ ...]
    （将定的展添加到 Xlink。）

    cm ^xlink -^dr|--^deleterules <Xlink_路径> <展>[ ...]
    （从 Xlink 中除定的展。）

    Xlink_路径          是当前工作区中将装入（建 Xlink ）
                        或已装入（ Xlink ）
                        已接的存的目。
    更集格            程存中的完整目更集
                        格。
                        此格定了已接的存的工作区中
                        加了个版本和分支。
                        （使用 'cm ^help ^objectspec' 可一了解更集
                        格。）
    格              程存中的完整格。
                        （使用 'cm ^help ^objectspec' 可一了解
                        格。）
    分支格              程存中的完整分支格。
                        此格使用指定的分支所指向的
                        当前更集。（使用 'cm ^help ^objectspec' 可
                        可一了解分支格。）
    -^e                  有的 Xlink 以更改目更集
                        格。
    -^s | --^show         示有所 Xlink 的信息。
    -^ar | --^addrules    将一个或多个展添加到所 Xlink。
    -^dr | --^deleterules 从所 Xlink 中除一个或多个展
                        。
    展     指定一个或多个展。个展
                        都是一分支/目分支：
                        ^br:/main/fix-^br:/main/develop/fix

：

    -^w                  表示 Xlink 是可写的。意味着
                        可以通分支自展功能来修改
                        目存的内容。
    -^rs                 相服器。此允建独立于
                        存服器的相 Xlink。，
                        在不同服器的制存中
                        建的 Xlink 将被自。

== CMD_HELP_XLINK ==
注：

    此命令将建指向定更集的 Xlink。默情况下将建一个
    只 Xlink。意味着无法修改 Xlink 内部工作区中
    加的内容。了能更改通 Xlink 接的内容，
    建可写的 Xlink（使用 '-^w' ）。

    在 Xlink 的目更集，可以使用
    命令的化法。，唯一需要的参数是
    新的目更集。Xlink 的其余参数将不会
    被修改。

    分支自展：

    在任何可写 Xlink 接的存（'-^w' ）中行更改，
    需要在目存中建一个新分支。新分支的
    名称基于存中定的出
    分支。了定要使用的分支的名称，遵循以下：

    1) 系会目存中是否存在具有
       相同全名的分支：
         - 如果存在，将此分支用作出分支。
         - 如果不存在，通以下方式建分支名称：
           - 通 Xlink 接的目更集的分支名称 + 出分支
             的短名称（最后一部分）。
           - 如果存在此分支，将此分支用作出分支。
           - 否，将建分支并将分支基置
             通 Xlink 接的更集。

    2) 在父存的分支中建 Xlink 的新版本，
        指向通 Xlink 接的存中的新更集。

    最后，完整的 Xlink 会与正版本中的
    最新更改保持同。

示例：

    cm ^xlink code\firstrepo / 1@first@localhost:8084
    （在当前工作区的文件 'firstrepo' 中建一个 Xlink，
    在其中将装入存 'first' 中的更集 '1'。）

    cm ^xlink opengl\include /includes/opengl 1627@includes@localhost:8087
    （在当前工作区的目 'opengl\include' 中建只
     部分 Xlink，在其中，存 'includes' 的更集 '1627' 中的路径
     '/includes/opengl' 将作根目装入。意味着 '/includes/opengl'
     中的任何内容都将装入到 'opengl\include' 中，
    而存的其余部分将被忽略。）

    cm ^xlink -^w -^rs code\secondrepo / ^lb:LB001@second@localhost:8084
    （在当前工作区的文件 'secondrepo' 中建一个
     可写的相 Xlink，在其中将装入存 'second' 中的
     'LB001'。）

    cm ^xlink code\thirdrepo / 3@third@localhost:8087 ^br:/main-^br:/main/scm003
    （在当前工作区的文件 'thirdrepo' 中建一个 Xlink，
    在其中将装入存 'third' 中的更集 '3'。）

    cm ^xlink -^e code\secondrepo ^br:/main/task1234@second@localhost:8084
    （通接存 'second' 中的分支 'main/task1234'，
     Xlink 'code\secondrepo' 以更改目存。）

    cm ^xlink --^show code\thirdrepo
    （示 Xlink 'code\thirdrepo' 的信息，包括其展
     （如果存在））。

    cm ^xlink -^ar code\secondrepo ^br:/main-^br:/main/develop ^br:/main/fix-^br:/main/develop/fix
    （将个展添加到 Xlink 'code\secondrepo'。）

    cm ^xlink -^dr code\secondrepo ^br:/main/fix-^br:/main/develop/fix
    （从 Xlink 'code\secondrepo' 中除展）。

== CMD_USAGE_AUTOCOMPLETE ==
用法：

    cm ^autocomplete ^install
    （在 shell 中安装 'cm' 命令功能。）

    cm ^autocomplete ^uninstall
    （从 shell 中卸 'cm' 命令功能。）

    cm ^autocomplete --^line <shell_行> --^position <光位置>
    （返回自建，以便将“shell_行”插入
     “光位置”。此命令并非供最用使用，但里
     此命令是考到可能需要展所 shell 的
    自支持。）

    shell_行      当求自由用
                    写入 shell 的行。
                    在 Bash 中，此位于 COMP_LINE 境量中。
                    在 PowerShell 中，位于 $wordToComplete 量中。
    光位置 求自的
                    光位置。
                    在 Bash 中，此位于 COMP_POINT 境量中。
                    在 PowerShell 中，位于 $cursorPosition 量中。

== CMD_DESCRIPTION_CONFIGURECLIENT ==
Configures the Unity VCS client for the current machine user to work with a default server.

== CMD_USAGE_CONFIGURECLIENT ==
用法：
    cm ^configure [--^language=<language> --^workingmode=<mode> [AuthParameters] 
                 --^server=<server> [--^port=<port>]] [--^clientconf=<clientconfpath>]

    --^language          可用言：
                        en (English)
                        es (Spanish)

    --^workingmode       可用的用/安全工作模式：
                        NameWorkingMode (Name)
                        NameIDWorkingMode (Name + ID)
                        LDAPWorkingMode (LDAP)
                        ADWorkingMode (Active Directory)
                        UPWorkingMode (User and password)
                        SSOWorkingMode (Single Sign On)

    AuthParameters      身参数（用于 ^LDAPWorkingMode 和 ^UPWorkingMode）：
                        --^user=<user>
                        --^password=<password>

                        Single Sign On parameters (only for ^SSOWorkingMode):
                        --^user=<user>
                        --^token=<token>

    --^server            Unity VCS server IP / address

    --^port              Unity VCS server port
                        (port optional for Cloud servers)

    --^clientconf        用于建配置文件的文件路径（可）
                        参数可以是完整路径、文件名或目。

                        示例：

                        --^clientconf=c:/path/to/myclient.conf
                        （指定的路径将用于建客端配置文件）

                        --^clientconf=myclient.conf
                        （将使用默配置目中的文件 myclient.conf）

                        --^clientconf=c:/exisitingDirectory
                        （将使用指定目中的默文件名 client.conf）

== CMD_HELP_CONFIGURECLIENT ==
注：

    The cm ^configure command cannot be used on Cloud Edition or DVCS Edition of Unity VCS.
    Use 'plastic --configure' instead.

示例：

    cm ^configure 
    (runs the interactive Unity VCS client configuration command)

    cm ^configure --^language=^en --^workingmode=^LDAPWorkingMode --^user=^jack --^password=^01234 \
                 --^server=^plastic.mymachine.com --^port=^8084
    (configures the Unity VCS client with the specified parameters and creates the 'client.conf'
    configuration file in the default directory).
    （Cloud 服器的可端口）

    cm ^configure --^language=^en --^workingmode=^NameWorkingMode --^server=^plastic.mymachine.com \
                 --^port=^8084 --^clientconf=^clientconf_exp.conf
    (configures the Unity VCS client with the specified parameters and creates the 'client.conf'
    configuration file in the specified path).
    （Cloud 服器的可端口）
